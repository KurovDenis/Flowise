# Specification: Добавление Enumeration Management Tools в MCP Evently

**Дата:** 2025-01-27  
**Статус:** Требуется реализация  
**Приоритет:** Высокий

---

## Цель

Добавить в MCP Evently 6 новых tools для управления перечислениями, которые уже реализованы в Evently API после спецификации `001-enumeration-management`, чтобы агенты Flowise могли полноценно управлять перечислениями и их значениями.

## Текущее состояние

В MCP Evently реализованы только операции чтения и создания перечислений (`get_enumerations`, `get_enumeration`, `create_enumeration`), тогда как в API уже доступны операции обновления и удаления для перечислений и их значений.

## Требуемые изменения

Добавить 6 новых MCP tools: `update_enumeration`, `delete_enumeration`, `add_enumeration_value`, `update_enumeration_value`, `delete_enumeration_value`, и `batch_delete_enumeration_values`. Для каждого tool необходимо создать схему валидации входных данных в `validation.ts`, определить tool в методе `setupHandlers()` с описанием параметров, и реализовать обработку вызова в `handleCallTool()` через соответствующие API endpoints. Все операции должны поддерживать проверку зависимостей и корректную обработку ошибок, включая частичное удаление при массовых операциях.

## Критерии успеха

После реализации агенты Flowise смогут полностью управлять жизненным циклом перечислений: создавать, обновлять, удалять перечисления, а также управлять их значениями с автоматическим разрешением конфликтов порядка и проверкой зависимостей перед удалением.

---

**Объем работы:** ~200-300 строк кода, 6 новых tools, 6 схем валидации  
**Время реализации:** 2-4 часа
